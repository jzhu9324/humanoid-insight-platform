## 🔧 完成的优化项目

### 1. 修复核心问题 ✅

#### 问题 1: 网站不自动更新
**原因**: commit message 带 `[skip ci]` 标签阻止了 deploy workflow

**解决方案**:
- 移除 `[skip ci]` 标签
- 现在每次数据更新都会自动触发网站重新部署

**相关 commit**: `9635172`

#### 问题 2: Papers/Company-News/Reports 列表不更新
**原因**: index.md 是静态文件，手动维护

**解决方案**:
- 创建 `scripts/generate_indexes.py` 自动生成 index 页面
- 在 workflow 中添加自动生成步骤
- 每次数据收集后自动更新所有 index

**相关 commit**: `a95f410`

#### 问题 3: 重复收录旧新闻
**原因**:
- Google News RSS 返回的旧新闻被重复抓取
- 去重只在保存时进行，效率低

**解决方案**:
- 在收集阶段就过滤已收录的链接
- 检查过去 14 天的历史记录
- 没有新内容时不生成空文件

**相关 commit**: `ed6483f`, `d3e07ec`

#### 问题 4: sources.json 有 merge conflict
**原因**: Git merge 冲突未解决

**解决方案**:
- 手动解决 merge conflict
- 验证 JSON 语法正确性

**相关 commit**: `ed6483f`

### 自动化流程

**每天 00:00 UTC（北京时间 08:00）**
```
1. 收集论文（Semantic Scholar + HuggingFace + arXiv）
2. 收集公司动态（28 个 RSS 源）
3. 收集微信文章
4. 生成日报
5. 更新 index 页面
6. 去重过滤（检查 14 天历史）
7. AI 分析和摘要
8. 自动 commit + push
9. 触发网站部署
```

**每天 12:00 UTC（北京时间 20:00）**
```
1. 收集公司动态（第二次）
2. 收集微信文章（第二次）
3. 更新 index 页面
4. 去重过滤
5. AI 分析和摘要
6. 自动 commit + push
7. 触发网站部署
```

### 核心功能

- ✅ **智能去重**: 检查过去 14 天历史，在收集阶段就过滤
- ✅ **多源聚合**: 28 个 RSS 源 + Google News 全网搜索
- ✅ **AI 分析**: MiniMax M2.5 自动生成摘要
- ✅ **自动部署**: 数据更新后自动触发网站重新部署
- ✅ **错误处理**: 单个源失败不影响整体收集
- ✅ **自动 Index**: 自动生成所有列表页面
- ✅ **在线配置**: 通过网页直接编辑数据源
